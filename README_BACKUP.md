# บันทึกการปรับปรุงระบบ (NWT Document Submission System Backup)
**วันเวลาที่สำรองข้อมูล:** 16 มิถุนายน 2569 เวลา 11:30 น.

รายละเอียดสรุปงานและการปรับปรุงความปลอดภัยและอินเตอร์เฟส (UI) ทั้งหมดที่ดำเนินการมีดังนี้:

---

## 1. การเชื่อมต่อระบบส่งข้อมูลการแจ้งเตือน Realtime (Pusher Integration)
- **ระบบหลังบ้าน (Backend)**:
  - สร้างไฟล์ [PusherService.php](file:///e:/edocs-spmnara%2016.6.69/public_html/services/PusherService.php) ทำงานผ่าน cURL PHP REST API โดยไม่ต้องติดตั้ง dependencies ภายนอก
  - เพิ่ม Event `new-request` บนช่อง `admin-channel` เมื่อมีการส่งใบคำขอใหม่ในหน้าหลัก
  - เพิ่ม Event `new-message` บนช่อง `request-{request_no}` เมื่อประชาชนหรือเจ้าหน้าที่พิมพ์ข้อความในแชทโต้ตอบ
  - เพิ่ม Event `status-updated` บนช่อง `request-{request_no}` เมื่อเจ้าหน้าที่เปลี่ยนสถานะการดำเนินงานของเอกสาร
- **ระบบหน้าบ้าน (Frontend JS)**:
  - เชื่อมต่อตัวรับข้อความ Pusher JS ในหน้าแดชบอร์ด หน้าติดตามคำขอ และหน้ารายละเอียดคำขอ
  - ระบบจะเล่นเสียงแจ้งเตือนและยิง Toast Alert / Reload ในหน้าแดชบอร์ดทันทีเมื่อมีรายการยื่นคำขอใหม่เข้ามาแบบ Realtime

---

## 2. การลดยอดความเสี่ยงความปลอดภัย (Security Improvements)
- ย้ายข้อมูลเชื่อมต่อทั้งหมดออกจากตัวแปรค่าคงที่ใน Source Code เพื่อป้องกันข้อมูลรั่วไหลเมื่อนำเข้าระบบ Git
- สร้างไฟล์ [.env](file:///e:/edocs-spmnara%2016.6.69/public_html/.env) สำหรับจัดเก็บ Database Credentials, SMTP Mail Credentials และ Pusher Keys
- สร้างไฟล์ [.env.example](file:///e:/edocs-spmnara%2016.6.69/public_html/.env.example) เพื่อเป็นตัวอย่างฟอร์แมตตั้งค่า
- สร้างไฟล์ [.gitignore](file:///e:/edocs-spmnara%2016.6.69/.gitignore) เพื่อยกเว้นไฟล์ `.env` และเอกสาร PDF ส่วนบุคคลไม่ให้ถูกอัปโหลดขึ้น Git
- พัฒนาคลาสโหลด [Env.php](file:///e:/edocs-spmnara%2016.6.69/public_html/app/config/Env.php) ทำการดึงค่าผ่านฟังก์ชัน `Env::get(...)` พร้อมใส่ค่า default สำรอง

## 3. การแก้ไขข้อผิดพลาดทางอินเตอร์เฟสและโครงสร้าง Layout (UI/UX Overhaul)
- **แก้ไขข้อผิดพลาด HTML Nesting ในหน้า Dashboard**:
  - ตรวจพบแท็ก `</div>` ไม่ครบที่กล่องกราฟเส้น ทำให้แถวที่สอง KPIs คิวตาราง และระบบคัดกรองทั้งหมด ยัดตัวเองเข้าไปอยู่ในคลาส `col-md-6` ส่งผลให้หน้าจอเลื่อนเพี้ยน มีพื้นที่ว่างสีเทาเปล่าๆ ด้านล่างลึกถึง 2500px และตารางถูกบีบไปทางขวา
  - ทำการปิดแท็ก HTML ทั้งหมดให้สมบูรณ์ ทำให้ Grid Layout แสดงผลตรงตามโครงสร้างปกติ
- **แก้ไข Sidebar สูงเกินขนาดในอุปกรณ์เคลื่อนที่ (Mobile Sidebar Height Bug)**:
  - แก้ไขคลาส CSS ของแถบเมนูข้างในหน้าผู้ดูแลระบบที่ตั้งค่า `min-height: 100vh` ตลอดเวลา ซึ่งบน Mobile ที่ Sidebar กองอยู่ด้านบน มันจะผลักหน้าจอยื่นคำขอเปล่าๆ ลงไปข้างล่างเต็มจอ
  - ปรับปรุงให้เป็น `min-height: auto` บนจอ Mobile และ `min-height: 100vh` เฉพาะบน Desktop เท่านั้นผ่าน Media Queries
- **การปรับปุ่มบนเมนูหลักสำหรับโทรศัพท์ (Responsive Navigation Button)**:
  - ปรับตำแหน่งปุ่ม "ติดตามคำขอ" และ "เข้าสู่ระบบ" ใน Navbar ให้แสดงผลแบบ Stack (แนวตั้ง) บนหน้าจอเล็ก และแนวนอนบนหน้าจอขนาดใหญ่
- **เพิ่มตัวช่วยวิเคราะห์การทำงานโหลด (Fullscreen Loading Spinner)**:
  - เพิ่ม CSS/JS คลาสกล่อง Overlay หน้าจอเบลอและหมุนโหลดอัตโนมัติเมื่อประชาชนหรือเจ้าหน้าทีกดบันทึกฟอร์ม/ยื่นเอกสาร/ส่ง OTP ป้องกันการกดซ้ำระหว่างประมวลผลเซิร์ฟเวอร์
