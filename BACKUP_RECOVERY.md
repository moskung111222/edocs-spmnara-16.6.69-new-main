# คู่มือการสำรองข้อมูลและการกู้คืนระบบ (Backup & Recovery Manual)

คู่มือฉบับนี้อธิบายถึงขั้นตอนการสำรองข้อมูล (การทำงานอัตโนมัติของสคริปต์รายวัน) และขั้นตอนการทำงานแบบควบคุมมือ (Manual) เพื่อทำการกู้คืนระบบเมื่อเกิดภัยพิบัติหรือระบบล่ม (Disaster Recovery)

---

## 1. รายละเอียดระบบสำรองข้อมูล (Backup Specifications)

ระบบจะสำรองข้อมูลที่สำคัญ 2 ส่วน ลงไปยังโฟลเดอร์สำรองข้อมูลลับภายนอกพื้นที่เผยแพร่เว็บ (เช่น `/home/USERNAME/private_storage/backups/`) ทุกวันเวลา 23:00 น. โดยเก็บย้อนหลังสูงสุด 30 วัน:
- **ฐานข้อมูล**: จะสำรองในรูปแบบคำสั่งโครงสร้างและข้อมูลดิบไฟล์ SQL บีบอัด (.sql)
- **ไฟล์แนบเอกสาร PDF**: จะสำรองในรูปแบบการเข้ารหัสรวบรวมไฟล์ Zip Archive (.zip)

---

## 2. การสั่งสำรองข้อมูลด้วยตนเอง (Manual Trigger Backup)

หากมีความจำเป็นต้องทำการสำรองข้อมูลด่วนด้วยตนเอง (เช่น ก่อนการปรับปรุงเซิร์ฟเวอร์ หรือปรับปรุงโปรแกรม) สามารถทำได้โดย:

1. เชื่อมต่อเซิร์ฟเวอร์ผ่านระบบ **SSH**
2. ไปยังไดเรกทอรีที่ติดตั้งไฟล์สำรองข้อมูล (เช่น `/home/USERNAME/`)
3. รันคำสั่งควบคุมผ่าน PHP CLI:
   ```bash
   php cron_backup.php
   ```
4. ระบบจะพิมพ์ขั้นตอนการส่งออกข้อมูลและบันทึกไฟล์สำรองข้อมูลชุดปัจจุบันลงยังโฟลเดอร์ backups ทันที

---

## 3. ขั้นตอนการกู้คืนข้อมูลระบบ (System Recovery Procedure)

เมื่อระบบขัดข้องหรือเกิดกรณีฐานข้อมูลและไฟล์แนบชำรุดเสียหาย ให้ดำเนินงานกู้คืนระบบตามลำดับขั้นตอนดังนี้:

### ขั้นตอนที่ 3.1: ดาวน์โหลดไฟล์สำรองข้อมูล
1. เปิดเมนู **File Manager** บน DirectAdmin
2. เข้าไปยังพื้นที่จัดเก็บไฟล์แบ็คอัพ: `/home/USERNAME/private_storage/backups/`
3. ตรวจหาและเลือกไฟล์แบ็คอัพชุดที่มีความสมบูรณ์และล่าสุด (พิจารณาจากชื่อ วันที่ และขนาดไฟล์ที่สอดคล้อง)
   - ไฟล์ฐานข้อมูล: `db_backup_YYYY-MM-DD_HH-II-SS.sql`
   - ไฟล์เอกสารแนบ: `files_backup_YYYY-MM-DD_HH-II-SS.zip`

### ขั้นตอนที่ 3.2: กู้คืนฐานข้อมูล (Database Recovery)
1. เปิด **phpMyAdmin** หรือเชื่อมต่อฐานข้อมูลผ่าน CLI
2. แนะนำให้ทำการ Drop ตารางทั้งหมดในฐานข้อมูล `edocs_spmnara` เพื่อป้องกันข้อมูลโครงสร้างเก่าตีกัน
3. ไปที่แท็บ **Import (นำเข้า)** -> คลิกเลือกไฟล์ -> เลือกไฟล์ `db_backup_xxxx.sql` ที่ดาวน์โหลดไว้
4. ตรวจสอบให้แน่ใจว่ารหัสอักขระเป็น `utf8mb4` จากนั้นกด **Go (เริ่ม)**
5. ตรวจสอบคิวข้อมูลประวัติทั้งหมดของตารางและดัชนีคีย์หลักจะต้องกลับมาสมบูรณ์

### ขั้นตอนที่ 3.3: กู้คืนไฟล์แนบเอกสาร (Files & Documents Recovery)
1. ไปที่ไดเรกทอรีเก็บเอกสารใช้งานจริง: `/home/USERNAME/private_storage/documents/`
2. เคลียร์ไฟล์เก่าที่ชำรุดเสียหายทิ้ง
3. อัปโหลดไฟล์สำรองข้อมูล `files_backup_xxxx.zip` เข้ามาไว้ที่ไดเรกทอรีนี้
4. สั่งแตกไฟล์บีบอัด (Extract / Unzip) ผ่านเมนูใน DirectAdmin File Manager หรือผ่าน SSH command:
   ```bash
   unzip files_backup_xxxx.zip -d /home/USERNAME/private_storage/documents/
   ```
5. ตรวจสอบให้แน่ใจว่าลบไฟล์ zip สำรองออกจากโฟลเดอร์นี้เพื่อไม่ให้สับสน
6. ตั้งค่าการเข้าสิทธิ์ของโฟลเดอร์และไฟล์กลับเป็น 700 หรือ 750 เช่นเดิม:
   ```bash
   chmod 750 /home/USERNAME/private_storage/documents
   chmod 640 /home/USERNAME/private_storage/documents/*
   ```

### ขั้นตอนที่ 3.4: ตรวจสอบความพร้อมการกู้คืน
1. ลองเปิดระบบหน้าติดตามสถานะคำขอประชาชน เพื่อทดลองดึงคีย์เลขที่ขอเดิมดู
2. ลองทดสอบเปิดไฟล์หลักฐานผ่าน PDF.js ในหน้าจอเจ้าหน้าที่ หากสามารถอ่านเอกสารประกอบการพิจารณาได้ครบถ้วน แสดงว่าขั้นตอนกู้คืนเสร็จสิ้นสมบูรณ์
