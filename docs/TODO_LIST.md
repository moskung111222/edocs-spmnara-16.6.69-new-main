# TODO List (รายการงานที่ต้องทำและการพัฒนาระบบต่อ)

เอกสารฉบับนี้สรุปรายการพัฒนาฟังก์ชันที่ยังขาดอยู่ รวมถึงการขยายประสิทธิภาพของระบบ โดยจัดเรียงลำดับความสำคัญ (Priority) เพื่อใช้เป็นแนวทางพัฒนาต่อเนื่องสำหรับทีมงานไอทีหรือนักพัฒนาที่จะรับช่วงต่อ

## 1. ลำดับความสำคัญสูง (High Priority)

### 1.1 ระบบจัดการบัญชีผู้ใช้เจ้าหน้าที่ปฏิบัติงาน (Officer Management UI)
* **ปัญหาปัจจุบัน:** ยังไม่มีส่วนควบคุมสร้างบัญชีผ่านหน้าเว็บ แอดมินต้องเพิ่มข้อมูลลงตาราง `officers` ด้วยตนเองผ่าน PHPMyAdmin หรือสคริปต์ SQL
* **ความต้องการ:**
  * พัฒนาหน้าระบบการจัดการบัญชีผู้ใช้งาน (เพิ่ม/แก้ไข/ลบ/เปลี่ยนสถานะการใช้งานบัญชี)
  * ระบบเปลี่ยนรหัสผ่านเจ้าหน้าที่และระบบกู้คืนบัญชี (Reset Password)
  * การบังคับการเข้ารหัสผ่านใหม่และตรวจสอบสิทธิ์การใช้งานของ Role ในหน้าจัดการบัญชี
* **ไฟล์ที่ต้องพัฒนาเพิ่ม:**
  * View: `public_html/views/admin/officers.php`
  * Controller: คลาสเพิ่มฟังก์ชัน `AdminController::manageOfficers()`
  * Model: `App\Models\Officer` เพื่อรวบรวม CRUD SQL query

### 1.2 หน้าสำหรับตั้งค่าประเภทคำขอเอกสารและรายการเอกสารแนบ (Request Types & Checklists UI)
* **ปัญหาปัจจุบัน:** ประเภทคำขอถูกฝังไว้ในตาราง `request_types` การจะแก้ไขรายการเอกสารแนบ (Checklist JSON) จำเป็นต้องมีความรู้ SQL
* **ความต้องการ:**
  * พัฒนาหน้าจอแสดงรายการประเภทคำขอ และแก้ไขฟิลด์ข้อมูลต่าง ๆ
  * พัฒนาอินเทอร์เฟซสร้างอินพุตเอกสารแนบแบบไดนามิก เพื่อให้เจ้าหน้าที่สามารถเพิ่มเอกสารแนบที่จำเป็นลงในเช็คลิสต์ประเภทคำขอนั้น ๆ ได้ทันที
* **ไฟล์ที่ต้องพัฒนาเพิ่ม:**
  * View: `public_html/views/admin/request_types.php`
  * Controller: คลาสเพิ่มฟังก์ชัน `AdminController::manageRequestTypes()`

## 2. ลำดับความสำคัญปานกลาง (Medium Priority)

### 2.1 บูรณาการระบบ SMS Gateway สำหรับส่งรหัส OTP (SMS OTP Gateway Integration)
* **ปัญหาปัจจุบัน:** การรับรหัสยืนยันตัวตน (OTP) มีเฉพาะอีเมล หากประชาชนพิมพ์อีเมลผิดหรือเมลตกหล่นไปอยู่ในถังขยะ (Spam) จะไม่สามารถติดตามสถานะคำขอได้เลย
* **ความต้องการ:**
  * เชื่อมต่อ SMS API Gateway (เช่น THAISMS, SMS-Gateway-TH ฯลฯ)
  * ส่งรหัส OTP 6 หลักเข้าเบอร์โทรศัพท์มือถือที่บันทึกไว้ในตอนยื่นคำขอ ควบคู่ไปกับการส่งผ่านอีเมล
  * พัฒนาโครงสร้างการบันทึกสถานะ SMS ในฐานข้อมูลเพื่อจับคู่บันทึกความล้มเหลว (Failed SMS logs)
* **ไฟล์ที่ต้องพัฒนาเพิ่ม:**
  * Service: `App\Services\SmsService` สำหรับเชื่อมโยง API ปลายทาง
  * Controller: เพิ่มเงื่อนไขการส่งผ่าน SMS ใน `RequestController::create()`

### 2.2 ระบบกู้คืนฐานข้อมูลผ่านหน้าควบคุม Admin (Backup Restore UI)
* **ปัญหาปัจจุบัน:** ระบบมีสคริปต์ `cron_backup.php` สำหรับสำรองฐานข้อมูลและแนบซิปไฟล์เอกสาร PDF ไว้แล้ว แต่หากต้องการกู้คืนไฟล์ แอดมินต้องสั่งผ่าน Terminal หรือ phpMyAdmin ด้วยตนเอง
* **ความต้องการ:**
  * สร้างหน้าสำหรับให้สิทธิ์เฉพาะผู้ดูแลระบบหลัก (Admin Role เท่านั้น) ในการเรียกดูรายการไฟล์ซิปแบ็คอัพทั้งหมดบนเซิร์ฟเวอร์
  * เพิ่มปุ่มเพื่อดาวน์โหลดไฟล์แบ็คอัพเก็บไว้ภายใน และปุ่มกู้คืนฐานข้อมูลคืนค่ากลับโดยด่วน (Restore Database from ZIP)
* **ไฟล์ที่ต้องพัฒนาเพิ่ม:**
  * View: `public_html/views/admin/backups.php`
  * Controller: ฟังก์ชัน `AdminController::manageBackups()` และ `AdminController::restoreBackup()`

## 3. ลำดับความสำคัญต่ำ (Low Priority)

### 3.1 ระบบส่งออกรายงานสถิติเป็น Excel / PDF (Export to Excel/PDF)
* **ความต้องการ:**
  * พัฒนาหน้าส่งออกข้อมูลเชิงสรุปรายเดือน รายปี หรือรายแผนก
  * ดึงสถิติตัวชี้วัดการดำเนินการ (KPI) ของเจ้าหน้าที่แต่ละรายส่งออกเป็นเอกสารใบรับรองไฟล์ PDF หรือตาราง Excel
* **ไฟล์ที่ต้องพัฒนาเพิ่ม:**
  * Controller: เมธอดในการจัดทำสเปรดชีต (เช่น ใช้ PhpSpreadsheet หรือสร้างไฟล์ CSV ส่งออกแบบปลอดภัย)

### 3.2 การทำระบบสองภาษา (Localization / Multi-language Support)
* **ความต้องการ:**
  * ปรับเปลี่ยนหน้าสาธารณะ (หน้าเว็บประชาชน ยื่นคำขอ ตรวจสอบ OTP และแชตสนทนา) ให้รอบรับภาษาอังกฤษสำหรับติดต่อชาวต่างชาติหรือบุคลากรต่างพื้นที่
  * สร้างไฟล์แปลภาษา (Language Pack) เป็นอาร์เรย์เก็บคำศัพท์
* **ไฟล์ที่ต้องพัฒนาเพิ่ม:**
  * App Config: สร้างคลาสแปลภาษา `App\Config\Lang`
