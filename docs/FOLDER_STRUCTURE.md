# Folder Structure (โครงสร้างโฟลเดอร์โครงการ)

เอกสารฉบับนี้สรุปความหมายและหน้าที่ของโฟลเดอร์และไฟล์ต่างๆ ในโครงสร้างโปรเจกต์ยื่นคำขอเอกสารออนไลน์ สพม.นราธิวาส เพื่ออำนวยความสะดวกในการทำความเข้าใจการจัดแผนผังโค้ด (Code Architecture) สำหรับนักพัฒนาที่จะนำระบบไปพัฒนาต่อ

---

## แผนผังโฟลเดอร์ภาพรวม (Overall Project Directory Map)

```text
/ (Root Directory)
├── docs/                           <-- เอกสารคู่มือระบบ (โครงสร้าง, ความปลอดภัย, ฐานข้อมูล)
├── private_storage/                <-- เก็บไฟล์ PDF ที่ประชาชนอัปโหลด (ไว้นอก Web Root)
│   └── documents/                  <-- โฟลเดอร์เก็บไฟล์ PDF เอกสารแนบหลักฐาน
├── migration.sql                   <-- ไฟล์สคีมาฐานข้อมูลและข้อมูลตั้งต้น (Seed Data)
├── cron_backup.php                 <-- ไฟล์สคริปต์สำรองข้อมูลฐานข้อมูลและไฟล์เอกสารรายวัน
├── router.php                      <-- ไฟล์ Router สำหรับการทดสอบรันด้วย Local Built-in Server
├── .gitignore                      <-- ไฟล์ระบุประเภทไฟล์ที่ไม่ต้องการนำขึ้น Git (เช่น .env, backup)
└── public_html/                    <-- ส่วนหน้าเว็บที่สามารถเข้าถึงได้จากภายนอก (Web Root)
    ├── .env                        <-- ไฟล์เก็บข้อมูลสำคัญและคอนฟิกูเรชัน (SMTP, DB, Pusher)
    ├── .env.example                <-- ไฟล์ตัวอย่างการตั้งค่าสภาพแวดล้อม
    ├── .htaccess                   <-- การตั้งค่า URL Rewriting สำหรับ Apache
    ├── index.php                   <-- Front Controller และ Router หลัก
    ├── api/                        <-- ส่วนบริการ RESTful API ส่งกลับข้อมูลเป็นรูปแบบ JSON
    ├── app/                        <-- โฟลเดอร์การตั้งค่าหลักของระบบ
    ├── assets/                     <-- ไฟล์ทรัพยากรหน้าเว็บ (CSS, JS, Fonts, Images)
    ├── controllers/                <-- คลาสประมวลผลคำขอ (Controllers)
    ├── middleware/                 <-- ตัวคัดกรองคำขอ (HTTP Middlewares ด้านความปลอดภัย)
    ├── models/                     <-- คลาสสำหรับเชื่อมโยงฐานข้อมูล (Models)
    ├── services/                   <-- บริการเสริมสนับสนุนระบบ (Services เช่น อีเมล, อัปโหลด, แจ้งเตือน)
    └── views/                      <-- เทมเพลตสำหรับหน้าจอแสดงผล (HTML/PHP Views)
```

## คำอธิบายรายโฟลเดอร์และไฟล์สำคัญ (Folder and File Specifications)

### 1. ไดเรกทอรีรูท (Root Directory)
* **`docs/`**: แหล่งรวบรวมไฟล์เอกสารประกอบระบบทั้งหมด (เช่นภาพรวมระบบ สถาปัตยกรรมระบบ คอนฟิกฐานข้อมูล และคู่มือการติดตั้งติดตั้ง)
* **`private_storage/`**: พื้นที่เก็บเอกสารแนบ PDF ลับของประชาชน เพื่อให้มั่นใจได้ว่าจะไม่สามารถดึงข้อมูลไปได้จากภายนอก (ป้องกันการเรียกผ่าน URL เว็บตรงๆ)
* **`migration.sql`**: สคริปต์ SQL สำหรับใช้สร้างตารางทั้ง 9 ตารางในระบบฐานข้อมูล และประกอบด้วยข้อมูลประเภทคำขอตั้งต้นรวมทั้งรหัสผ่านทดสอบของเจ้าหน้าที่
* **`cron_backup.php`**: ถูกออกแบบมาให้เรียกใช้ผ่าน Linux Cron Jobs ทุกวัน เพื่อดึงข้อมูลตารางและซิปโฟลเดอร์เอกสารแนบเก็บสำรองไว้แบบออฟไลน์
* **`router.php`**: สคริปต์ขนาดเล็กเพื่อประมวลผลจำลองการทำงาน URL Rewrite สำหรับนักพัฒนาที่รันด้วย PHP Local Development Server (`php -S localhost:8080 -t public_html router.php`)

### 2. ไดเรกทอรี `public_html` (พื้นที่เผยแพร่บนเว็บ)
เป็นพื้นที่เผยแพร่ที่ผูกเข้ากับโดเมนเนมหลัก ทำหน้าที่เป็นประตูบานแรกในการรับส่งคำขอบนเว็บ

* **`index.php`**: จุดเริ่มต้นการประมวลผลโปรแกรมทั้งหมด (Front Controller) ทำหน้าที่:
  1. ลงทะเบียน Class Autoloader (โหลดคลาสอัตโนมัติตาม Namespace `App\`)
  2. โหลดตัวแปรสภาพแวดล้อมจากไฟล์ `.env`
  3. ตั้งค่าการเข้ารหัสข้อมูล XSS โกลบอลและเริ่มระบบเซสชัน
  4. ทำหน้าที่ค้นเส้นทาง (Router) ส่งงานต่อให้ Controller ที่เกี่ยวข้อง
* **`.env`**: ไฟล์เก็บข้อมูลคีย์ลับสำหรับการเชื่อมต่อ ไม่ควรถูกอัปโหลดขึ้น Git (เก็บค่า DB_USER, DB_PASS, SMTP_USER, SMTP_PASS, PUSHER_SECRET ฯลฯ)
* **`.htaccess`**: กำหนดค่าการทำงานของ Apache Web Server บังคับใช้งาน Rewrite Engine เพื่อนำส่ง URL ทุกตัวที่ระบุเข้ามาให้ส่งผ่านไปให้ `index.php?route=...` ประมวลผลโดยประชาชนไม่เห็นชื่อไฟล์จริง

### 3. โฟลเดอร์ `public_html/api` (Web APIs)
ประมวลผลลัพธ์ย้อนกลับมาให้กับหน้าบ้าน (Frontend) เป็น JSON Array
* **`requests.php`**: ค้นหารายการคำขอเอกสารทั้งหมด (ใช้บนแดชบอร์ดเจ้าหน้าที่)
* **`status.php`**: บริการเปลี่ยนสถานะหรือแสดงสถานะล่าสุดของเอกสาร
* **`upload.php`**: รับไฟล์ อัปโหลดประมวลผลไฟล์ผ่านระบบรักษาความปลอดภัย และบันทึกลงพื้นที่ Private Storage

### 4. โฟลเดอร์ `public_html/app/config` (System Configs)
ประกอบด้วยคลาสตั้งค่าระบบภายใน Namespace `App\Config`
* **`Env.php`**: ตัวโหลดและอ่านค่าจากไฟล์ `.env` ไปลงทะเบียนในโปรแกรม
* **`Database.php`**: คลาสจัดการการเชื่อมต่อฐานข้อมูลดึงค่าผ่าน API ของ MySQLi
* **`config.php`**: ตั้งค่าประเภท URL ไซต์งาน, ค่าขนาดไฟล์อัปโหลดสูงสุด (10MB), นามสกุลไฟล์ที่อนุญาต และข้อความสถานะภาษาไทย
* **`mail.php`**: จุดคืนค่า SMTP สำหรับส่งผ่าน PHPMailer

### 5. โฟลเดอร์ `public_html/controllers` (Application Controllers)
ส่วนควบคุมและส่งต่องานควบคุมการไหลของระบบ (MVC - Controller Layer)
* **`AuthController.php`**: ควบคุมการลงชื่อเข้าใช้งานของเจ้าหน้าที่ ตรวจสอบรหัสผ่านผ่าน Hash Bcrypt ล็อกเซสชันใหม่เพื่อป้องกัน Session Fixation และควบคุมการออกจากระบบ
* **`RequestController.php`**: จัดการฝั่งผู้รับบริการ (ประชาชน): แสดงประเภทคำขอ, ประมวลผลแบบฟอร์มบันทึก, ส่ง/ยืนยัน OTP ยืนยันตัวตน, แสดงผลหน้าติดตามงาน และสตรีมไฟล์เพื่อดาวน์โหลดอย่างปลอดภัย
* **`AdminController.php`**: จัดการฝั่งผู้ตรวจรับ: คำนวณ KPIs แสดงแดชบอร์ดสถิติ กรองงาน มอบหมายผู้ดูแล และระบบห้องแชตคุยประสานงาน

### 6. โฟลเดอร์ `public_html/middleware` (Request Filters)
สกัดกั้นและตรวจสอบสิทธิ์ก่อนส่งต่อการรันโปรแกรม
* **`AuthMiddleware.php`**: ยืนยันสิทธิ์เซสชันของเจ้าหน้าที่ (กรอง Role) และตรวจสอบสถานะยืนยันตัวตน OTP ของประชาชนผู้ติดตามงาน
* **`CsrfMiddleware.php`**: ตรวจสอบโทเค็นรักษาความปลอดภัย เพื่อป้องกันการส่งฟอร์มปลอมจากภายนอก

### 7. โฟลเดอร์ `public_html/models` (Database Models)
ติดต่อเรียกค้นข้อมูลจากฐานข้อมูลโดยใช้ไวยากรณ์ SQL (MVC - Model Layer)
* **`Applicant.php`**: จัดการประวัติประชาชนผู้รับบริการ
* **`Attachment.php`**: จัดการข้อมูลที่เก็บความสัมพันธ์ของเอกสารแนบ
* **`Request.php`**: ตัวคำนวณและประมวลผลดึงคำขอ ประวัติการเปลี่ยนสถานะ คำนวณยอดสถิติแสดงผล และระบบส่งข้อความ

### 8. โฟลเดอร์ `public_html/services` (Business Services)
ตัวช่วยแบ่งเบาภาระในคลาสควบคุมหลัก
* **`MailService.php`**: ผูกร่วมกับ PHPMailer เพื่อจัดส่งอีเมลยืนยันตัวตน (OTP) และแจ้งเตือนเมื่อคำขอมีความคืบหน้า
* **`PusherService.php`**: บริการ Realtime Trigger เพื่อสะท้อนข้อความแชตขึ้นหน้าจอเจ้าหน้าที่ทันทีโดยไม่ต้องกดรีเฟรชเบราว์เซอร์
* **`UploadService.php`**: ตรวจสอบไฟล์อัปโหลดอย่างละเอียด (ขนาด, ชนิดเนื้อหาไบนารี PDF) และบันทึกจัดเก็บเข้าไดเรกทอรีส่วนตัวอย่างปลอดภัย

### 9. โฟลเดอร์ `public_html/views` (Visual templates)
โครงสร้างหน้ากาก UI แสดงผล (MVC - View Layer) แยกเป็นหน้าตาของประชาชน (`public/`) และหน้าจอทำงานของเจ้าหน้าที่ (`admin/`)
* **`views/public/`**: หน้ายื่นคำขอ (`create.php`), หน้าจอแรก (`index.php`), หน้ากรอก OTP คัดกรองความปลอดภัย (`verify.php`), และหน้าตรวจสอบติดตามงานสนทนาประสานงาน (`track.php`)
* **`views/admin/`**: หน้าจอผู้ปฏิบัติงานหลักแสดง KPIs แผนภูมิสรุปยอดกรองงาน (`dashboard.php`), หน้าต่างตรวจสอบรายละเอียดคำขอจัดส่งแชตและมอบหมายงาน (`request_detail.php`), และหน้าต่างลงชื่อเข้าใช้งานระดับพรีเมียม (`login.php`)
* **`assets/css/style.css`**: คอนฟิกการออกแบบระบบ CSS (Design System) ทั้งปุ่ม เลย์เอาต์ แดชบอร์ด และธีมสีพรีเมียมเข้ม (Dark Mode)
