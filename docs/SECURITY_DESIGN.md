# Security Design (การออกแบบความปลอดภัยของระบบ)

เอกสารฉบับนี้อธิบายถึงสถาปัตยกรรมการรักษาความปลอดภัย (Security Architecture) ที่ถูกนำมาใช้ออกแบบและพัฒนาในโครงการระบบยื่นคำขอเอกสารออนไลน์ สพม.นราธิวาส เพื่อปกป้องข้อมูลประชาชนและป้องกันการโจมตีระบบตามมาตรฐาน OWASP

---

## 1. การป้องกัน CSRF (Cross-Site Request Forgery)
ระบบใช้กลไกการสุ่มและตรวจสอบ Token สำหรับทุกการทำงานที่เป็นธุรกรรมเปลี่ยนสถานะข้อมูล (State-Changing Operations) ผ่าน HTTP POST:

* **การสร้างและจัดเก็บ Token:** ดำเนินการโดย [CsrfMiddleware](file:///e:/edocs-spmnara%2016.6.69/public_html/middleware/CsrfMiddleware.php) เมื่อเริ่มต้นเซสชัน ระบบจะสุ่มสร้างโทเค็นขนาด 32 ไบต์ ด้วย `bin2hex(random_bytes(32))` และเก็บไว้ใน `$_SESSION['csrf_token']`
* **การแทรกโทเค็นในฟอร์ม:** หน้ากากอินเทอร์เฟซ (Views) จะเรียกใช้ตัวช่วยสร้างฟิลด์ `CsrfMiddleware::getHtmlField()` เพื่อฝังแท็ก `<input type="hidden" name="csrf_token" value="...">` ลงในฟอร์ม
* **การตรวจสอบโทเค็น (Verification):**
  * เมื่อมีการส่งคำขอแบบ `POST` ระบบจะเรียกใช้งาน `CsrfMiddleware::validatePostRequest()` เพื่อเปรียบเทียบโทเค็นที่ได้จากผู้ส่ง (ใน POST payload หรือ HTTP Header `X-CSRF-TOKEN`) กับโทเค็นที่อยู่ในเซสชันด้วยฟังก์ชันที่ป้องกันการโจมตีแบบ Timing Attack: `hash_equals()`
  * หากโทเค็นไม่ตรงกัน ระบบจะปฏิเสธคำขอด้วยรหัสข้อผิดพลาด **403 Forbidden** ทันที

---

## 2. การป้องกัน XSS (Cross-Site Scripting)
เพื่อป้องกันไม่ให้ผู้ไม่หวังดีฝังโค้ดสคริปต์ที่เป็นอันตราย (Malicious Scripts) เข้ามาแสดงผลในระบบ:

* **Escaping Helper:** ระบบประกาศฟังก์ชันระดับโกลบอลชื่อ `esc()` ใน [index.php](file:///e:/edocs-spmnara%2016.6.69/public_html/index.php):
  ```php
  function esc($string) {
      return htmlspecialchars($string ?? '', ENT_QUOTES, 'UTF-8');
  }
  ```
* **การบังคับใช้ในส่วนแสดงผล:** ทุกจุดที่มีการดึงข้อมูลจากอินพุตผู้ใช้งานหรือฐานข้อมูลมาแสดงผลในไฟล์เทมเพลตวิว จะถูกห่อหุ้มด้วยฟังก์ชัน `esc()` เสมอ เช่น `<?= esc($request['full_name']) ?>` หรือในระบบแชตสนทนา
* **JSON Encoding:** สำหรับข้อมูลที่เป็น JSON คอนโทรลเลอร์จะส่งออกผ่าน `json_encode()` ซึ่งทำการแปลงอักขระพิเศษอย่างปลอดภัย

---

## 3. ความปลอดภัยของเซสชัน (Session Security)
การควบคุมพฤติกรรมเซสชันของผู้ใช้งานเพื่อป้องกันสิทธิ์ถูกขโมย (Session Hijacking / Session Fixation):

* **การตั้งค่าพารามิเตอร์คุกกี้เซสชัน:** มีการกำหนดคุณลักษณะความปลอดภัยของเซสชันเมื่อเริ่มต้นระบบด้วย `session_set_cookie_params()`:
  * **HttpOnly:** กำหนดเป็น `true` เพื่อห้ามไม่ให้สคริปต์ฝั่งไคลเอนต์ (เช่น JavaScript `document.cookie`) เข้าถึงตัวระบุเซสชันได้ ลดความเสี่ยงกรณีเกิด XSS
  * **Secure:** กำหนดเป็น `true` โดยอัตโนมัติหากเซิร์ฟเวอร์รันอยู่บน HTTPS เพื่อบังคับส่งคุกกี้ผ่านช่องทางเข้ารหัสเท่านั้น
  * **SameSite:** กำหนดเป็น `'Lax'` เพื่อป้องกันการแชร์คุกกี้ข้ามเว็บไซต์ที่อาจเกิดภัยคุกคามประเภท CSRF
* **การทำลายฟิกเซชัน (Session Fixation Countermeasure):** เมื่อเจ้าหน้าที่ลงชื่อเข้าใช้งานสำเร็จใน [AuthController](file:///e:/edocs-spmnara%2016.6.69/public_html/controllers/AuthController.php) ระบบจะสั่งงาน `session_regenerate_id(true)` เพื่อยกเลิกเซสชันไอดีเดิมและสร้างไอดีตัวใหม่ขึ้นมาทันที

## 4. ความปลอดภัยในการอัปโหลดและดาวน์โหลดไฟล์ (Upload & Download Security)
เนื่องจากระบบเปิดให้ประชาชนทั่วไปอัปโหลดไฟล์หลักฐาน จึงจำเป็นต้องมีมาตรการคัดกรองที่เข้มงวดเพื่อป้องกันการอัปโหลดมัลแวร์หรือการโจมตีผ่านไฟล์เว็บเชลล์:

* **การจำกัดประเภทและขนาดไฟล์:**
  * กำหนดประเภทไฟล์ที่อนุญาตเฉพาะ PDF เท่านั้น (`application/pdf`)
  * จำกัดขนาดไฟล์สูงสุดไม่เกิน 10MB ต่อหนึ่งไฟล์
* **การตรวจสอบประเภทไฟล์จริง (MIME Type Verification):** ตรวจสอบไฟล์ด้วยการอ่านหัวไฟล์ผ่านฟังก์ชันของ PHP ใน [UploadService](file:///e:/edocs-spmnara%2016.6.69/public_html/services/UploadService.php) โดยใช้ `finfo_file()` ไม่ตรวจสอบเพียงแค่นามสกุลไฟล์ที่ผู้ใช้อัปโหลดมา
* **การเปลี่ยนชื่อไฟล์และแยกพื้นที่จัดเก็บ (Safe Location Isolation):**
  * เมื่อผ่านการตรวจสอบ ไฟล์จะถูกเปลี่ยนชื่อเป็นแบบสุ่ม (เช่น แฮช UUID หรือชื่อสุ่ม)
  * ย้ายไฟล์ไปเก็บไว้นอกพื้นที่เว็บรูทที่ [private_storage/documents/](file:///e:/edocs-spmnara%2016.6.69/private_storage/documents/) ซึ่งเข้าถึงผ่านเบราว์เซอร์ตรงๆ ไม่ได้ (เช่น เรียกผ่าน URL ตรงๆ จะขึ้น 404 หรือ Access Denied เสมอ)
* **การควบคุมการดาวน์โหลด (Secure File Streaming):** เมื่อเจ้าหน้าที่ขอดาวน์โหลดเอกสาร ไฟล์จะถูกอ่านและสตรีมมิ่งผ่านคลาส [RequestController::download()](file:///e:/edocs-spmnara%2016.6.69/public_html/controllers/RequestController.php) ซึ่งจะทำการตรวจสอบสิทธิ์ของผู้ใช้งานในเซสชันก่อนส่งออกข้อมูลไบนารีและตั้งค่า Header ป้องกัน Path Traversal (`basename()`)

## 5. การควบคุมการเข้าถึงระบบและบันทึกการใช้งาน (Access Control & Audit Trail)
* **การตรวจสอบสิทธิ์ควบคุม (Middleware Enforcement):**
  * เมธอดและอินเทอร์เฟซผู้ดูแลระบบทั้งหมดจะถูกป้องกันผ่าน [AuthMiddleware::requireOfficer()](file:///e:/edocs-spmnara%2016.6.69/public_html/middleware/AuthMiddleware.php)
  * หน้าติดตามสถานะและแชตของประชาชนจะถูกป้องกันผ่าน `AuthMiddleware::requireTrackingAuth($requestNo)` เพื่อให้แน่ใจว่าประชาชนเข้าถึงได้เฉพาะเลขที่คำขอที่ตนเองได้รับรหัส OTP เท่านั้น
* **การบันทึก Audit Logs สำหรับสืบสวนย้อนหลัง:**
  * ทุกกิจกรรมสำคัญ เช่น การลงชื่อเข้า/ออกระบบ การแก้ไขสถานะคำขอ การมอบหมายงาน หรือการประมวลผลระบบ จะถูกบันทึกลงตารางฐานข้อมูล `audit_logs` ผ่านเมธอด `RequestService::logAudit()` โดยอัตโนมัติ
  * บันทึกข้อมูลประกอบด้วย: ไอดีผู้ทำรายการ, ประเภทผู้ใช้ (ประชาชน/เจ้าหน้าที่/ระบบ), กิจกรรมที่ทำ, โมดูลระบบ, IP Address จริงของผู้ใช้งาน, และข้อมูล User Agent ของเบราว์เซอร์
