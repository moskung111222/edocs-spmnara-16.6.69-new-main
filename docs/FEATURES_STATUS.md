# Features Status (สถานะการพัฒนาคุณสมบัติระบบ)

เอกสารฉบับนี้สรุปสถานะการพัฒนาฟังก์ชันการทำงานต่างๆ ของระบบยื่นคำขอเอกสารออนไลน์ สพม.นราธิวาส จากโครงสร้างโค้ดและตารางฐานข้อมูลปัจจุบัน เพื่อให้นักพัฒนาสามารถวางแผนและต่อยอดระบบได้อย่างชัดเจน

---

## 1. คุณสมบัติที่เสร็จสมบูรณ์แล้ว (Completed Features)

### 1.1 ฝั่งบริการประชาชน (Citizen-Facing Features)
* **หน้าลงทะเบียนคำขอใหม่ (Document Request Form):**
  * ระบบเลือกประเภทคำขอยื่นใบแทนแทนใบสุทธิ (HS), ขอทรานสคริปต์ (TR), หรือตรวจสอบวุฒิ (ED)
  * ฟอร์มรวบรวมข้อมูลตามที่กำหนด และอัปโหลดไฟล์ PDF ประกอบคำขอตามตารางเช็คลิสต์เอกสารแนบแบบไดนามิก
* **การพิสูจน์ตัวตนด้วย OTP (Passwordless OTP Verification):**
  * ป้องกันการติดตามข้อมูลจากบุคคลอื่น ด้วยการสุ่มรหัส OTP 6 หลัก ส่งเข้ากล่องอีเมลผู้ขอเพื่อนำมาตรวจสอบสิทธิ์ยืนยันตัวตน
* **หน้าติดตามสถานะและสนทนา (Request Tracking & Live Chat):**
  * แสดงประวัติขั้นตอนสถานะคำขอ (ยื่นคำขอแล้ว, ตรวจสอบ, อนุมัติ ฯลฯ) แบบเรียลไทม์
  * ระบบแชตสนทนาตอบกลับกับเจ้าหน้าที่และประชาชนแบบสองทิศทาง (Bi-directional Chat) ร่วมกับการแจ้งเตือนแบบเรียลไทม์ (Pusher Realtime Event)
  * อนุญาตให้อัปโหลดเอกสาร PDF เพิ่มเติมได้โดยตรงผ่านห้องแชต เฉพาะในกรณีที่คำขออยู่ในสถานะแก้ไขเอกสาร (`need_info`)

### 1.2 ฝั่งเจ้าหน้าที่ปฏิบัติงานและผู้รับผิดชอบ (Officer-Facing Features)
* **ระบบลงชื่อเข้าใช้งานของเจ้าหน้าที่ (Officer Authentication):**
  * หน้าจอล็อกอินแยกส่วนสำหรับเจ้าหน้าที่ ตรวจสอบความปลอดภัยระดับรหัสผ่านด้วย Bcrypt และเพิ่มสิทธิ์ป้องกันเซสชัน (Session Fixation Countermeasure)
* **แดชบอร์ดสรุปผลและรายงานผู้บริหาร (Dashboard & Executive KPIs):**
  * รายงานผลสรุปยอดคำขอแต่ละสถานะ แสดงแนวโน้มคำขอรายเดือนและแยกประเภท
  * คำนวณ KPI ประสิทธิภาพการประมวลผล (เช่น จำนวนคำขอที่ดำเนินการเสร็จสิ้นในแต่ละเดือน, อัตราความรวดเร็วเฉลี่ยในการปิดงานคำขอรายเดือน)
* **ระบบจัดการคำขอและการมอบหมายงาน (Request Manager & Assignment):**
  * ระบบตรวจสอบรายละเอียดคำขอ ตรวจดูเอกสารหลักฐานของประชาชนผ่านตัวกรองดูข้อมูลภายในระบบ
  * หัวหน้ากลุ่มงานหรือแอดมินสามารถทำการมอบหมายงาน (Assign) ให้กับเจ้าหน้าที่รายคน หรือทำการยกเลิกการมอบหมายงานได้
  * กล่องแชตและบันทึกข้อมูลภายใน (Internal Notes) ซึ่งเห็นเฉพาะกลุ่มเจ้าหน้าที่เพื่อประสานงานภายใน

### 1.3 ความปลอดภัยและระบบสนับสนุน (Security & Operations)
* **การป้องกันความปลอดภัยเว็บ (Web Security Implementations):**
  * ระบบป้องกัน CSRF บนโพสต์ฟอร์มทั้งหมด และการเข้ารหัส XSS ด้วยฟังก์ชัน `esc()` ตลอดจนระบบควบคุมคุกกี้ระดับปลอดภัยสูงสุด (HttpOnly, SameSite=Lax, Secure Cookie)
* **ระบบจัดเก็บไฟล์ PDF ปลอดภัย (Secure Private Storage):**
  * เก็บไฟล์ PDF ของประชาชนไว้นอกรูทของเว็บ และเข้าถึงผ่านฟังก์ชันดาวน์โหลดตรวจสอบสิทธิ์ของคอนโทรลเลอร์เท่านั้น ป้องกันภัยคุกคาม Path Traversal
* **ระบบบันทึกความเคลื่อนไหว (Audit Logging):**
  * จัดเก็บรายละเอียดผู้ใช้งานที่บันทึกข้อมูล วันเวลาการแก้ไข ไอพีของเครื่อง และตัวระบุเบราว์เซอร์ ลงในตาราง `audit_logs` ทุกครั้ง
* **ระบบสำรองข้อมูลประจำวันอัตโนมัติ (Automated Daily Backups):**
  * สคริปต์ `cron_backup.php` สำหรับรันบนเซิร์ฟเวอร์เพื่อซิปฐานข้อมูลพร้อมแนบไฟล์เอกสารเก็บแยกอย่างปลอดภัย

## 2. คุณสมบัติที่กำลังดำเนินการ / ปรับปรุง (In Progress Features)
* **การปรับปรุงความสวยงามและการตอบสนองหน้าเว็บ (Responsive UI Polish):**
  * การตรวจสอบและแก้ไของค์ประกอบเลย์เอาต์หน้าบ้าน (Frontend HTML/CSS) เพื่อให้รองรับการทำงานสมบูรณ์แบบบนเบราว์เซอร์โทรศัพท์มือถือ (Mobile Responsive) และหน้าตากระชับ ทันสมัย Premium ธีมสีเข้ม (Dark Mode) บนหน้าแดชบอร์ดทำงาน

## 3. คุณสมบัติที่ยังไม่ได้พัฒนา / ขยายผลต่อยอดได้ (Missing Features / Roadmap)
* **ระบบจัดการบัญชีผู้ใช้เจ้าหน้าที่ผ่าน UI (Officer Account Manager):**
  * *สถานะปัจจุบัน:* ข้อมูลเจ้าหน้าที่ต้องจัดการแก้ไขผ่านชุดข้อมูล Seed ในตาราง `officers` โดยตรงหรือเขียน SQL เอง
  * *สิ่งที่แนะนำ:* พัฒนาอินเทอร์เฟซ UI หน้าจัดการ เพิ่ม/ลบ/แก้ไขข้อมูลเจ้าหน้าที่ รวมถึงตัวเลือกในการเปลี่ยนรหัสผ่านและการกำหนดบทบาท (Role) ของเจ้าหน้าที่
* **ระบบจัดการประเภทคำขอเอกสารผ่าน UI (Request Types & Checklists Manager):**
  * *สถานะปัจจุบัน:* ประเภทคำขอและรายการเอกสารที่จำเป็น (Doc Checklist) ต้องแก้ไขผ่านข้อมูลประเภทคำขอในตาราง `request_types`
  * *สิ่งที่แนะนำ:* พัฒนาหน้าจอตั้งค่าตัวจัดประเภทคำขอ (เช่น ปิด/เปิด บริการยื่นคำขอบางรายการ, เพิ่มประเภทคำขอเอกสารใหม่ หรือแก้ไขรายการเอกสารที่จำเป็นต้องแนบแบบเรียลไทม์)
* **ระบบกู้คืนฐานข้อมูลผ่านหน้าเว็บผู้ดูแลระบบ (Backup Restore UI):**
  * *สถานะปัจจุบัน:* หากต้องการนำข้อมูลสำรองกลับมาใช้ ต้องกระทำผ่านคำสั่ง Command CLI หรือนำเข้าผ่าน phpMyAdmin ด้วยตนเอง
  * *สิ่งที่แนะนำ:* พัฒนาหน้าการจัดการไฟล์แบ็คอัพเพื่อดาวน์โหลดคัดลอก หรือกดสั่งงานนำเข้าคืนค่า (Restore) ข้อมูลเก่าคืนกลับระบบจากหน้าเว็บของผู้ดูแลระบบได้ทันที
* **การส่งรหัสผ่าน OTP สำรองผ่านทาง SMS (SMS OTP Gateway Integration):**
  * *สถานะปัจจุบัน:* ส่งรหัส OTP และแจ้งเตือนสถานะความคืบหน้าผ่านช่องทาง Mail SMTP เท่านั้น
  * *สิ่งที่แนะนำ:* นำเข้า SMS API Gateway เพื่อใช้เป็นช่องทางหลักหรือช่องทางสำรองในการแจ้งเตือนรหัสความปลอดภัยเข้ามือถือประชาชนโดยตรง
