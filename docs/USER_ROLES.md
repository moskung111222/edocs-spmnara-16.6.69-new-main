# User Roles and Permissions (สิทธิ์และบทบาทผู้ใช้งาน)

เอกสารฉบับนี้อธิบายถึงบทบาทผู้ใช้งาน (User Roles) และการควบคุมการเข้าถึงระบบยื่นคำขอเอกสารออนไลน์ของ สพม.นราธิวาส (NWT Document Submission System) โดยแบ่งกลุ่มผู้ใช้งานออกเป็น 4 ระดับตามหน้าที่และความรับผิดชอบจริงที่ปรากฏในซอร์สโค้ด

---

## 1. ประชาชนทั่วไป / ผู้ยื่นคำขอ (Citizen / Applicant)
เป็นผู้รับบริการที่เข้ามายื่นคำขอและติดตามสถานะคำขอของตนเอง

### การพิสูจน์ตัวตน (Authentication)
* เข้าใช้งานหน้าเว็บสาธารณะโดยไม่ต้องใช้รหัสผ่าน (Passwordless) สำหรับการยื่นคำขอใหม่
* สำหรับการติดตามสถานะคำขอ (Tracking View) ระบบใช้ระบบ **OTP (One-Time Password)** ส่งไปยังอีเมลที่ระบุไว้ในคำขอ โดยมีรหัสผ่านชั่วคราวอายุ 5 นาที เพื่อเข้าตรวจสอบข้อมูลเฉพาะคำขอของตนเองเท่านั้น (พิจารณาจากรหัสคำขอ `request_no`)

### สิทธิ์การทำงาน (Permissions)
* **ยื่นคำขอใหม่ (Submit Request):** เลือกประเภทคำขอ กรอกข้อมูลฟอร์ม และอัปโหลดไฟล์ PDF ตามรายการเอกสารที่กำหนด (Doc Checklist)
* **ยืนยันตัวตน (Verify OTP):** ยืนยันรหัส OTP เพื่อสร้าง Session สำหรับสิทธิ์ในการติดตามสถานะ
* **ติดตามสถานะ (Track Status):** เรียกดูข้อมูลรายละเอียดคำขอและประวัติสถานะปัจจุบันของตนเองได้แบบเรียลไทม์
* **พูดคุยสอบถาม (Chat Interface):** รับส่งข้อความกับเจ้าหน้าที่ผ่าน Chat Box เฉพาะคำขอของตนเอง (มีการแจ้งเตือนแบบเรียลไทม์ด้วย Pusher)
* **อัปโหลดเอกสารเพิ่มเติม (Upload Additional Files):** สามารถอัปโหลดไฟล์เอกสารเพิ่มเติมได้ **เฉพาะเมื่อคำขออยู่ในสถานะ `need_info` (ขอข้อมูลเพิ่มเติม/แก้ไขเอกสาร) เท่านั้น** โดยเมื่ออัปโหลดสำเร็จแล้ว สถานะของคำขอจะถูกเปลี่ยนกลับเป็น `submitted` (ยื่นคำขอแล้ว) โดยอัตโนมัติ

---

## 2. เจ้าหน้าที่ปฏิบัติงาน (Staff)
เป็นข้าราชการหรือเจ้าหน้าที่กลุ่มส่งเสริมการจัดการศึกษา สพม.นราธิวาส ที่มีหน้าที่หลักในการตรวจสอบ ตรวจรับ และประมวลผลคำขอเอกสาร

### การพิสูจน์ตัวตน (Authentication)
* ลงชื่อเข้าใช้งานผ่านระบบด้วย Username และ Password (ตรวจสอบโดยการเทียบค่าแฮช Bcrypt จากตาราง `officers`) ผ่านเส้นทาง `/admin/login`

### สิทธิ์การทำงาน (Permissions)
* **ดูรายการคำขอ (View Dashboard):** เข้าถึงแดชบอร์ดแสดงข้อมูลสถิติภาพรวมจำนวนคำขอแยกตามสถานะ และแสดงรายการคำขอทั้งหมดในระบบ
* **ค้นหาและกรองข้อมูล (Search & Filter):** ค้นหาคำขอด้วยข้อความ กรองตามสถานะ หรือกรองตามเจ้าหน้าที่ผู้รับผิดชอบ
* **ดูรายละเอียดคำขอ (View Request Details):** ดูรายละเอียดของคำขอเอกสาร ไฟล์แนบทั้งหมด ประวัติการเปลี่ยนสถานะ และข้อความสนทนา
* **บันทึกข้อความภายใน (Private Notes):** ส่งข้อความสนทนาแบบภายใน (Internal Note) สำหรับประสานงานระหว่างเจ้าหน้าที่ด้วยกัน ซึ่งประชาชนทั่วไปจะไม่สามารถมองเห็นข้อความนี้ได้
* **ตอบกลับประชาชน (Chat with Citizen):** ส่งข้อความสนทนาหาประชาชนผู้ยื่นคำขอโดยตรง
* **ปรับปรุงสถานะคำขอ (Change Status):** สามารถเปลี่ยนสถานะของคำขอ (เช่น จาก `submitted` เป็น `received`, `in_review`, `need_info`, `pending_approval`, `approved`, `completed` หรือ `rejected`) พร้อมระบุเหตุผลในการปรับสถานะ
* **ดาวน์โหลดเอกสาร (Download Attachments):** ดาวน์โหลดไฟล์ PDF เอกสารแนบจากที่จัดเก็บส่วนตัวแบบปลอดภัย (Private Storage)

---

## 3. หัวหน้ากลุ่มงาน (Head)
หัวหน้ากลุ่มงานที่มีบทบาทควบคุมดูแลการทำงานภาพรวมของเจ้าหน้าที่ปฏิบัติงาน

### การพิสูจน์ตัวตน (Authentication)
* เข้าสู่ระบบผ่านเส้นทาง `/admin/login` โดยบัญชีผู้ใช้ในตาราง `officers` จะต้องมีค่าคอลัมน์ `role` เป็น `'head'`

### สิทธิ์การทำงาน (Permissions)
* **ได้รับสิทธิ์ของ Staff ทั้งหมด** (สิทธิ์ในการดูรายการ ค้นหา เปลี่ยนสถานะ สนทนา บันทึกข้อมูลภายใน และดาวน์โหลดเอกสาร)
* **มอบหมายงาน (Assign Officer):** สามารถระบุตัวเจ้าหน้าที่ผู้รับผิดชอบ (Assignee) ให้กับคำขอแต่ละรายการได้ โดยเลือกจากรายชื่อเจ้าหน้าที่ในระบบ หรือยกเลิกการมอบหมายงานได้

---

## 4. ผู้ดูแลระบบ (Admin)
ผู้ดูแลระบบไอที (System Administrator) ที่ทำหน้าที่ดูแลโครงสร้างพื้นฐาน ความปลอดภัย และบัญชีผู้ใช้งาน

### การพิสูจน์ตัวตน (Authentication)
* เข้าสู่ระบบผ่านเส้นทาง `/admin/login` โดยบัญชีผู้ใช้ในตาราง `officers` จะต้องมีค่าคอลัมน์ `role` เป็น `'admin'`

### สิทธิ์การทำงาน (Permissions)
* **ได้รับสิทธิ์ของ Staff และ Head ทั้งหมด** (รวมถึงการมอบหมายงานและการเปลี่ยนสถานะคำขอ)
* **เข้าถึงแดชบอร์ดผู้บริหาร (Executive KPIs Dashboard):** เรียกดูข้อมูลรายงานสถิติ และ KPI การประมวลผลคำขอ (เช่น ระยะเวลาเฉลี่ยในการดำเนินการ หรือคำขอที่ค้างท่อ)
* **บันทึกประวัติการใช้งาน (Audit Logs Viewer):** สามารถเข้าถึงข้อมูลประวัติการทำงาน (Audit Logs) ทั้งหมดในระบบ เพื่อตรวจสอบย้อนหลังการเข้าใช้งานและการแก้ไขข้อมูลของทั้งประชาชน เจ้าหน้าที่ หัวหน้า และระบบ

---

## ตารางสรุปสิทธิ์การเข้าใช้งานเชิงหน้าที่ (Function-based Permission Matrix)

| หน้าที่การทำงาน | Citizen | Staff | Head | Admin | ไฟล์/เมธอดควบคุม (Code Enforcement) |
| :--- | :---: | :---: | :---: | :---: | :--- |
| ยื่นคำขอใหม่ (Create Request) | ✓ | - | - | - | `RequestController::create()` |
| ยืนยันตัวตน OTP (Verify OTP) | ✓ | - | - | - | `RequestController::verify()` |
| ติดตามสถานะ (Track Request) | ✓ | - | - | - | `RequestController::track()`, `AuthMiddleware::requireTrackingAuth()` |
| เรียกดูแดชบอร์ด (Dashboard) | - | ✓ | ✓ | ✓ | `AdminController::dashboard()`, `AuthMiddleware::requireOfficer()` |
| ดูรายละเอียดคำขอ (Detail View) | - | ✓ | ✓ | ✓ | `AdminController::requestDetail()`, `AuthMiddleware::requireOfficer()` |
| เปลี่ยนสถานะคำขอ (Change Status) | - | ✓ | ✓ | ✓ | `AdminController::requestDetail()`, `api/status.php` |
| สนทนากับประชาชน (Chat) | ✓ | ✓ | ✓ | ✓ | `api/upload.php`, `AdminController::requestDetail()` |
| บันทึกข้อความภายใน (Internal Note) | - | ✓ | ✓ | ✓ | `AdminController::requestDetail()`, `Request::getMessages()` |
| มอบหมายงาน (Assign Officer) | - | - | ✓ | ✓ | `AdminController::requestDetail() -> assign_officer` |
| ดูประวัติการใช้งาน (Audit Logs) | - | - | - | ✓ | `RequestService::logAudit()` |
